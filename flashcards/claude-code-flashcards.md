# Claude Code Configuration & Workflows Flashcards

### Front
What is `CLAUDE.md` best for?

### Back
Always-on project conventions and instructions that Claude should see every session.

### Front
When should plan mode be used instead of direct execution?

### Back
When the task is complex, risky, or requires understanding the full scope before making changes.

### Front
Why use session resumption or forking?

### Back
To recover from context bloat or try alternative approaches without losing valuable context.

### Front
How do hooks help in Claude Code workflows?

### Back
They run deterministic automation outside the model loop—intercepting calls, blocking actions, or normalizing results.

### Front
What belongs in `CLAUDE.md` versus a skill?

### Back
`CLAUDE.md` is for always-on project conventions (file patterns, build commands, coding standards). Skills are for reusable workflows or commands.

### Front
When should you create a custom skill versus using prompts?

### Back
Create a skill for reusable knowledge or workflow instructions that you'll use across multiple sessions or tasks.
